# 技能名稱：Ingest (知識攝取與增量編譯)

## 🎯 觸發時機
當我提供 `RAW/` 資料夾中的新檔案路徑，或要求處理新資料時。

## 🛠️ 執行步驟 (Agent Workflow)

### 步驟 1：附件攔截與互動 (若為附件檔案)
- 若目標檔案位於 `RAW/attachments/`，**請立刻暫停動作**。
- 向我提問：「請問這個檔案的作用是什麼？他是屬於哪個專案或與哪間公司/聯絡人有關？」。
- 等待我回答後，再將我提供的脈絡資訊與附件整合，繼續執行步驟 2。

### 步驟 2：深度閱讀、價值萃取與「狀態判定」
- 閱讀指定檔案內容。
- **佔位檔判定 (Stub Determination)**：若讀取的 `RAW/` 檔案內容為全空，或字數極少（例如只有標題、隨手記下的零星單字），請將其判定為「佔位檔（Stub）」。請根據「檔案名稱」（例如人名）以及我給予的上下文提示，推斷其屬性（聯絡人、公司或專案）。
- 萃取核心要素：專案名稱、聯絡人/公司、進度、里程碑與我的貢獻。
- **專案狀態判定（重要）**：判斷文件中提及的專案，其當前狀態為何（🟢 執行中、🟡 暫停/擱置、🔵 週期性/維護中、⚫ 已結案、或是 🚀 舊案重啟）。

### 步驟 3：Wiki 頁面編譯與更新 (需含 YAML 與時間軸)
- **路徑導向**：
    - 新專案建立於 `Wiki/專案追蹤/`。
    - 新人物建立於 `Wiki/聯絡人/`。
    - 新公司建立於 `Wiki/公司資料/`。
- **YAML 標記**：新建頁面必須加入 `origin: AI-generated`。針對判定為「佔位檔」的頁面，**必須**額外加入 `status: stub` 標籤。更新既有頁面時請更新 `last_updated` 時間。
- **內容處理**：
  - 針對佔位檔頁面，在內文中寫入初步推斷的資訊（例如：此為內部同仁，詳細職稱與聯絡方式待補充）。
  - **專案時間軸追蹤**：若為「舊案重啟」或「週期性重複執行」的專案，**絕對不可覆寫舊有紀錄**。請在該專案頁面中建立新的時間戳記標題（例如：`### [YYYY-MM-DD] 專案重啟 / 第二期導入`），並將新進度寫在下方，保留完整的歷史脈絡。
  - 將新萃取出的里程碑或個人貢獻，同步寫入 `Wiki/里程碑與價值沉澱/` 對應頁面。
- 寫入內文時，務必使用 `[[ ]]` 建立雙向連結。若為附件，以 `![[檔案名稱]]` 格式嵌入。

### 步驟 4：行政收尾與封存 (雙索引更新機制)
- **更新規則 (編碼要求)**：所有寫入 Wiki 的動作**必須確保為 UTF-8 編碼**。
- **更新目錄與日誌**：補入 `Wiki/index.md` 摘要；在 `Wiki/log.md` 新增操作紀錄。
- **更新附件總索引**：若處理的是附件，將資訊新增至 `Wiki/附件總索引.md` 的表格中。
- **更新專案總索引（核心動作）**：若本次處理涉及任何專案的建立或狀態改變，**務必**同步更新 `Wiki/專案總索引.md` 的表格。更新其「當前狀態」與「最後異動日」。
- **移動封存 (歸檔)**：處理完畢後，將原始檔案從 `RAW/` 移動至 `RAW/封存/` 底下的對應分類。**絕對不可修改原始檔案內容。**

### 步驟 5：完成回報
- 回報萃取的關鍵進度、專案狀態的變更、更新的 Wiki 節點，並確認檔案已成功移至封存區。
