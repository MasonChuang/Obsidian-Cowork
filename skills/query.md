# 技能名稱：Query (查詢與知識沉澱)

## 🎯 觸發時機
當我向你提問、要求總結專案進度、分析卡關原因、回顧個人貢獻，或進行跨文件分析時。

## 🛠️ 執行步驟 (Agent Workflow)
請嚴格遵守根目錄 `system.md` 的規範，並依序執行以下操作：

### 步驟 1：優先檢索全局索引
- **絕對禁止**一開始就盲目讀取所有檔案。
- 必須先讀取 `Wiki/index.md`，從中鎖定與我問題高度相關的實體、專案或里程碑頁面。

### 步驟 2：深度閱讀與綜合回答
- 根據步驟 1 鎖定的目標，沿著雙向連結 `[[ ]]` 讀取對應的 Wiki 詳細頁面。
- 綜合上述資訊給出精確、有條理且彰顯工作價值的回答。
- **必須標註來源**：在回答的段落後方，明確標示你的資訊引用自哪一個 Wiki 頁面（例如：依照 `[[專案A_追蹤]]` 的紀錄...）。

### 步驟 3：高價值知識沉澱 (核心動作)
- 自我評估你這次的產出：這是一份全新的跨專案洞察、一份完整的季度回顧、還是一套解決困難的 SOP？
- 如果產出具備重複利用或彰顯價值的潛力，請在回答的最後**主動詢問我**：「這份分析極具價值，請問是否需要將其獨立保存為新的 Wiki 頁面？」
- 若我回答同意，請將此內容以 Markdown 格式化，寫入 `Wiki/里程碑與價值沉澱/` 中，並執行步驟 4。

### 步驟 4：行政收尾 (更新導航與日誌)
- **編碼要求**：所有寫入與修改動作必須強制使用 **UTF-8**。
- 若在步驟 3 中建立了新頁面，請務必將其單行摘要與連結補入 `Wiki/index.md`。
- 在 `Wiki/log.md` 頂部新增操作紀錄。格式範例：`## [YYYY-MM-DD HH:MM] Query | 提問：[我的問題簡述] | 沉澱節點：[[新建立的Wiki頁面]] (若有)`。
