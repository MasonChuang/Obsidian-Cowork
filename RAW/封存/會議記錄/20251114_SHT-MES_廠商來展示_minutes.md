# 【會議記錄】世禾科技 MES 導入專案會議

## 1. 會議摘要 (Executive Summary)
本次會議主要由廠商進行 MES 系統功能演示（以航空維修案例為參考），並針對「世禾科技 MES 系統導入」之核心需求進行深度討論。重點涵蓋硬體儲存架構、圖片壓縮標準、中國大陸廠區顧問差旅費計費原則、以及最具爭議的 **17 碼編碼規則**。會議確立了未來系統開發將採密集訪談模式，並針對跨廠區資料同步提出初步方案。

## 2. 待釐清問題 (Queries)
- [00:41:00] **FLA 圖片存儲價值**：需確認 FLA 檢測圖片轉存至 MES Server 後，產線端除了「被動查閱」外，是否需與特定工單（Lot）自動關聯，以實現系統自動化刪除舊檔之功能。
- [00:48:00] **差旅費定義**：需釐清中國大陸出勤之「顧問費」計價標準，是指「純差旅天數」還是「實地工作天數」，且是否包含線上支援。
- [01:47:00] **財務對帳深度**：世巨李波提議之「PO 數量自動扣減與對帳」功能，涉及與現有財務系統對接，需由 IT 評估開發範疇是否超出原本 MES 範疇。

## 3. 詳細討論事項 (Discussion Details)

### A. 硬體與儲存規劃 (Infrastructure)
- **圖片壓縮**：目前產線拍攝圖片過大（6MB-9MB），計畫由系統自動壓縮至 300K-500K 之間。廠商將提供不同壓縮比的範例圖供產線與業務確認解析度是否達標。
- **儲存方案**：針對台積電（TSMC）FLA 檢測需求，建議建立排程自動將半年以上的圖片搬移至 Server 擴充硬碟，以釋放前端 PC 空間。

### B. 顧問費用與合約範疇 (Consulting Fees)
- **台灣地區**：合約內已包含標準教育訓練與安裝，不另收顧問費。若為「非計畫性」的額外需求（如重複培訓），則視情況報價。
- **中國大陸廠區**：凡涉及實地出勤，將依據合約外的人力報價計算（顧問費+出勤費）。若能將需求打包成獨立案子（CR），則可併入專案談判。

### C. 系統功能演示 (System Demo - Aviation Case)
- **權限管理**：展示可細分至「角色」、「部門」、「站點」的權限設定，支持 PDF 手冊線上查閱及異常「小鈴鐺」告警功能。
- **異常處理流程**：產線提報異常後，系統可自動發送 Email 通知主管與業務，並在首頁顯示待審核清單，確保審核履歷留存。

### D. 編碼邏輯討論 (Coding Rules - 17 Digits)
- **結構爭議**：顧崇誠顧問要求統一 17 碼編碼。會中討論包含：
  - **客戶碼 (4碼)**：台積、美光等大客戶給予固定碼，其餘小廠從 3000 開始流水編號。
  - **擴充性問題**：莊昀達提醒目前採用「純數字」編碼在高增長廠區（如台積電各 Fab）可能面臨位數不足，建議引入「英文字符（Alphanumeric）」以增加容量。
  - **製程編碼**：初步定義為 PVD、CVD、Etch 等類別，需解決同音或意義重複（如兩個 P 代表不同意義）之判讀問題。

### E. 生產管理需求 (Production Requirements)
- **耗材統計**：世巨提議在「圖層設定」中增加「原材料損耗欄位」（如砂量、膠帶、化學品），以便系統自動核算工件成本。
- **客戶名稱描述**：區分「世禾描述」與「客戶描述」。針對不同客戶對同工件之稱呼差異，系統需支援報表自動對應轉換。

## 5. 決策重點 (Key Decisions)
- **專案模式**：初期採取「密集訪談」模式，每兩週至少一次進度報告與原型（Prototype）展示，避免後期認知偏差。
- **資料同步**：跨廠區（大陸與台灣）資料庫不採即時同步，改採「Export/Import (匯入匯出)」方式同步編碼主檔，以規避跨境連線不穩問題。
- **編碼歸口**：所有新編碼原則由二廠（世禾台灣）統籌定義，各廠區遵照執行，嚴禁自行定義新編碼規則。
