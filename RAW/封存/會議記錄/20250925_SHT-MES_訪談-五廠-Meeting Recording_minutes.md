# 【會議記錄】世禾科技 MES 導入專案會議

## 1. 會議摘要 (Executive Summary)
本次會議針對世禾五廠導入 MES 系統之具體需求進行深度訪談，重點涵蓋**編碼原則建立**、**台積電(TSMC) B2B系統對接**、**生產工藝(Routing)細分程度**、**品質異常(MRB)處置流程**以及**紅白單(收送貨單)自動化產出**。達成共識將所有舊品視為新品重新編碼以確保資料唯一性，並確立品質異常之四種標準處置路徑。

## 2. 待釐清問題 (Queries)
- [Part 2, 00:33:45] 台灣與大陸廠區之專線頻寬僅 2M，IT 部門需評估系統上線後之負載能力，確認是否需提升頻寬。
- [Part 2, 01:36:30] 關於產線操作人員是否需在 MES 記錄「具體操作姓名」或「工時數據」，目前內部意見分歧（擔心增加負擔 vs. 追溯需求），需由高層最終核定。
- [Part 2, 03:27:00] 台積電各廠（台12、台15、台18）請款流程不一，部分堅持須回傳原紙本蓋章單據，業務部需進一步確認是否可推動全面電子化請款。

## 3. 詳細討論事項 (Discussion Details)

### 一、 基礎資料與編碼規則 (Master Data & Coding Rules)
- **編碼權責**：確認**業務部**負責客戶編碼與料件編碼；**研發部 (RD)** 負責定義生產工藝 (Routing) 編碼與工序內容。
- **舊資料處理策略**：由於舊系統 (TCS) 資料存在大小寫不一、命名重複（如同件物有八種名稱）等問題，會議決議**不匯入舊資料**。未來三個月內，所有進廠料件一律視為「新品」重新依照新規則編碼，以建立潔淨的資料庫。
- **客戶與部門綁定**：討論是否將「客戶碼」與「部門碼」合併。顧問建議透過系統下拉選單與關聯欄位處理，不一定非要擠在同一串編碼中，以維持編碼靈活性。

### 二、 生產工序與掃碼點位 (Routing & Scan Points)
- **工藝細分**：研發部提出工藝需細分（如泡酸、泡鹼、水刀、烘乾），但生產部擔心每步掃碼會影響效率。
- **決議方案**：MES 系統將支援「大站掃碼、細項記錄」模式。定義大站（如化學站）必須掃碼（進/出站），其內部的細節步驟（Step）則列在電子 Run Card 上供參考或勾選，可彈性設定是否強制掃碼。
- **機台對接**：未來規劃每一機台配置掃碼槍。目前採「彈性配置」，允許多台機台共用一處電腦工作站進行過站回報。

### 三、 台積電系統介接 (TSMC B2B Interface)
- **進貨資料**：生管目前需從台積電 SA360 (B2B 平台) 下載 Excel 資料。未來 MES 需具備「暫存區」功能，將 Excel 匯入後，人員掃描實物科號進行核對，核對正確後才正式轉為場內工單 (Work Order)。
- **RFID 應用**：台積電通箱 (Sharing Box) 上貼有 RFID 標籤，世禾需讀取標籤 ID 並與內部工單序號綁定，確保回貨時資料一致。

### 四、 品質異常 (MRB) 處置流程
當站點發現異常時，需暫停過站並通知 QC，由 QC 在系統開立異常單，並提供四種決策路徑：
1. **直接放行 (Release)**：異常不影響品質，記錄後繼續往下一站。
2. **退回重工 (Rework)**：退回前一站或指定工序重新執行。
3. **委外處理 (Outsource)**：轉入委外模組，發送郵件通知業務與客戶，並記錄委外廠商、單價與預計回廠時間。
4. **退回客戶 (RTV)**：無法修復，通知業務辦理退貨手續。

### 五、 行政報表與單據自動化
- **電子 Run Card**：系統根據工藝編碼自動產出包含照片、操作條件與追溯欄位的電子表單。
- **收送貨紅白單**：MES 需支援直接列印功能。針對特殊客戶需求（如台積電 15 廠），系統需能產出對應格式之單據供司機攜回蓋章。
- **API 接口**：資訊部提出 MES 需將完工資料傳送至 ERP 系統，作為會計部門請款之依據。

## 5. 決策重點 (Key Decisions)
- **[Decision 1]** 全面實施「新品化編碼」：不再延續舊有 TCS 系統之雜亂編碼，藉此專案重整物料主檔。
- **[Decision 2]** 異常處置權責歸屬：產線可開立異常提報，但「結案」與「處置路徑判定」權限嚴格限縮於品質部 (QC) 與業務部。
- **[Decision 3]** 記錄「清洗次數」與「告警設定」：系統需自動累計料件清洗次數，並在接近使用壽命時自動跳出 UI 警示，提醒業務部與客戶洽談報廢或更新。
