# 【會議記錄】世禾科技 MES 導入專案會議

## 1. 會議摘要 (Executive Summary)
本次會議由廠商進行 MQES (MES 系統) 之功能介紹與展示，主要針對清洗業設計的生產流程、SPC 串接、異常處理及報表產出進行探討。世禾科技專案團隊針對現行系統與公司大規模生產需求（如：逾期警示、跨廠區數據共享、照片上傳等）提出多項 Gap 分析。會議結論將朝向先進行「勞務測試階段」以釐清細部差異，再行評估客製化開發之成本與時程。

## 3. 詳細討論事項 (Discussion Details)

- **系統架構與基本資料設定**
  - **架構**：系統採 Client-Server 架構，提供圖示化介面進行料號與製程設定。
  - **階層化管理**：支援客戶別、部門別、料號之樹狀結構管理。
  - **製程彈性**：製程站點（大站）與工作步驟（細項）可自由定義，並支援多版本 (V1, V2, V3) 之圖層設定。
  - **Gap 需求**：**莊昀達**反映右側基本資料欄位目前不足，需支援更多客製化屬性欄位。

- **收料與公單管理 (Receiving & Lot Mgmt)**
  - **收料模式**：支援「圖層模式」及「公單模式」。若同批料號需走不同流程，需於源頭進行「拆單」。
  - **批號規則**：系統目前以料號作為 ID 判斷基準，不同零件需建立多張單據。
  - **Gap 需求**：**顧崇誠顧問**指出，世禾目前收件量大（單月逾萬件），現行「逐筆拆單」操作成本過高，且需強化「整套工件」的關聯性搜尋（如同一貨單號碼之批次管理）。

- **生產過站與異常處理 (Process Exceptions)**
  - **過站作業**：需人為點擊「開始」與「結束」以記錄工時。
  - **異常處理**：支援「續購 (Rework)」與「退件 (Return)」。異常發生時會發送 Mail 報警。
  - **Gap 需求**：**吳國偉廠長**反映，當前製程站清單無法直觀標示「異常回流單」，在一萬多件在製品中極難篩選，需增加視覺化 Highlight 標記。

- **SPC 與品質檢驗 (Quality Inspection)**
  - **串接機制**：MQES 根據「製程站名」與「料號」自動比對並調用 SPC 系統之規格上下限。
  - **警示功能**：量測值超出 SPEC 時會以「反紅」提示，但無自動發送 Alarm Mail 之機制。
  - **Gap 需求**：**吳國偉廠長**提出需支援「照片/檔案上傳」功能，以記錄進料或檢驗時的缺陷（Defect），並能查詢歷史損傷紀錄。

- **報表產出 (Reporting)**
  - **格式**：支援 COA (Excel) 與 ECoA (CSV) 套表匯出。
  - **Gap 需求**：**顧崇誠顧問**要求報表應能根據 Item Number 自動綁定對應範本，避免人工手動選擇路徑，並需支援多廠區數據之「合併報表」功能。

## 5. 決策重點 (Key Decisions)
- **測試先行原則**：因現行系統與世禾需求落差尚待具體量化，決定不直接購買，而是先針對「勞務測試費用」進行報價，由廠商派員進駐進行差異分析。
- **數據隔離與合併**：初期考量各廠區獨立資料庫運作，但最終必須具備跨廠區（如三廠、五廠）清洗次數累計與總部合併報表檢視之功能。
- **管理層面強化**：未來系統必須加入「逾期管理 (Overdue)」與「站點停留時間統計」功能，作為生產開會之管理依據。
