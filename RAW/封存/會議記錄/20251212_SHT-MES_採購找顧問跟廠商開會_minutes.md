# 【會議記錄】世禾科技 MES 導入專案會議

## 1. 會議摘要 (Executive Summary)
本次會議主要由採購部門、專案負責人 Mason 與顧崇誠顧問，針對 MES 系統導入專案與廠商進行合約細節、範疇界定及報價結構之對口。重點在於明確「客製化開發」之定位、六個廠區（台灣三廠、大陸三廠）的導入費用分攤、軟硬體拆分採購之會計原則，以及後續 UAT 與驗收之標準。會議確認將以集團化一套系統為目標，並落實生產履歷追蹤。

## 2. 待釐清問題 (Queries)
- **[00:20:53] 大陸導入人天費用**：需確認大陸現場導入之具體 Man-day 總量及顧問費（12,000/天）之預估上限，以利採購控帳。
- **[00:48:24] 驗收限制條款**：合約中「不可作為驗收作業限制條件」之描述與商業邏輯衝突，需法務與專案團隊進一步修訂文字。
- **[00:02:02 - Part2] OS版本**：需書面確認 Windows Server 最終採用版本為 2025 最新版，並同步更新硬體清單。

## 3. 詳細討論事項 (Discussion Details)

### A. 專案範疇與客製化定位
- **客製化模型**：顧顧問強調世禾非一般組裝業，無標準 BOM 損耗模型。本案採「量身定做 (Customization)」，而非標準版套用，旨在建立符合半導體清洗產業邏輯的系統。
- **管理重點**：現階段聚焦於「生產管理 (MES)」與「生產履歷」，而非全面的「設備管理 (EAM)」。
- **履歷追蹤**：因應台積電 (TSMC) 稽核需求，系統必須能追蹤 RunCard 在各站點（如噴砂、化學清洗）的作業員、設備編號及時間紀錄。

### B. 報價與合約拆分原則
- **軟硬體拆分**：採購 Winnie 要求廠商將報價拆分為「軟體」與「硬體」兩個 Item。
    - **會計原則**：硬體屬固定資產，落地即開始保固；軟體待驗收後起算保固。
    - **大陸支出**：差旅費（機票、食宿、交通）採實報實銷或定額補貼（如維也納酒店標準），須列入報價附件。
- **集團授權**：軟體授權涵蓋台灣（二、三、五廠）與大陸（世巨-合肥、世平-深圳、世正-成都）共六個廠區。

### C. 導入時程與驗收機制
- **專案時程**：預計 Kick-off 後 190 個工作天（約 9-10 個月）達成驗收。
- **驗收流程**：
    1. **UAT (使用者驗收測試)**：約一個月。
    2. **Parallel Run (平行作業)**：至少一個月，確保電子化流程與紙本一致且無 Bug。
- **維護保養 (MA)**：保固一年，後續 MA 費用預計按專案總價之一定百分比計算。

### D. 基礎資料準備 (Data Preparation)
- **編碼責任**：業務總部 (周宗治) 須主導不重複工件之編碼對應。Mason 提醒業務端目前進度落後，需將 14 萬筆再製品資料與系統 Index 對接。
- **模組化維護**：工藝流程、QA 代碼、COA 點位圖均採模組化設計（大/中/小類別），避免 user 頻繁手動輸入。

## 5. 決策重點 (Key Decisions)
- **硬體採購分開**：台灣硬體由專案統籌，大陸硬體（如 Server）由各廠自行按規範採購。
- **同步上線**：大陸世巨（合肥）將列為優先上線目標，與台灣廠區同步或緊隨其後進行 Test Run。
- **簽名機制**：暫定電子化 RunCard 須紀錄設備與 user ID，以符合 2026 年客戶稽核趨勢。
