# 【會議記錄】世禾科技 MES 導入專案會議

## 1. 會議摘要 (Executive Summary)
本次會議主要針對 MES 系統導入之「資料收集階段」進行進度追蹤與規格確認。重點討論工件基本資料（Static Data）的欄位增補、圖層工藝（Routing）的編碼簡化原則、以及品質檢驗（QC）項目的系統化需求。此外，針對世正（成都）廠區急需過渡期系統之需求，達成暫時複製世巨（合肥）現有系統之共識。

## 3. 詳細討論事項 (Discussion Details)

### A. 工件基本資料與編碼進度
- **進度要求**：預計於 2026 年 1 月底前完成客戶資料、工件資料、圖層（鍍層）資料、異常代碼及委外加工編碼。
- **欄位增補**：
  - 確定加入 **Part Number (P/N)** 與 **Serial Number (S/N)** 欄位。
  - 增設 **QC 檢測項目**：包含上限、下限、標準值、量測名稱，以及對應客戶 COA 的名稱。
  - 針對世巨需求，增加 **AOI Map 圖**、**內視鏡照片**、**CCD 放大照片**之連結或上傳欄位。
- **編碼原則**：客戶編碼採**四碼**制（如 1251 代表特定客戶之廠區+部門），由業務部統一編列後，各廠再透過 VLOOKUP 方式整合至再製品清單。

### B. 圖層工藝 (Routing) 與配方管理
- **簡化原則**：王志立經理提出將圖層區分為「十大流程」模板。顧問建議將「製程參數」（如時間、溫度、壓力）設為**變數**，避免因微小參數差異產生上千組工藝編碼。
- **配方保密**：RunCard 上僅顯示工藝代碼（如 2014A），後端資料庫連結實際配方（如 HF + H2O2 比例）。對於客戶稽核，將準備「對外版本」以解釋製程邏輯而不洩漏核心比例。
- **過站掃描**：確認品質檢驗（IQC/IPQC/OQC）亦需視為「大站」進行過站條碼掃描，以追蹤工件滯留時間與數據。

### C. 工程變更 (ECN/ECR) 系統化需求
- **追溯功能**：五廠吳國偉廠長建議系統需具備 ECN 追蹤功能。顧問裁示：系統將建立單獨的 ECN 模組，記錄變更時間、變更項目（圖層、參數或 COA 規格）及生效日期，以利後續溯源查詢。

### D. 世正（成都）廠區過渡期方案
- **需求**：成都廠因應客戶驗廠在即，無法等待三個月開發期。
- **決策**：擬採「系統克隆 (Ghost)」方案，將世巨（合肥）現有的維宜 (WEI YI) MES 伺服器完整複製一份至成都。
- **執行方式**：由世巨協助聯絡當地 IT 供應商，購買同規格硬體及 Windows Server 授權，並請莊昀達 (Mason) 協助技術把關。

## 5. 決策重點 (Key Decisions)
- **QC 模組強化**：MES 將針對 IQC, IPQC, OQC 分別開立 Table，並支援數值與圖片格式。
- **ECN 紀錄**：系統確定納入工程變更紀錄模組，捨棄單純紙本作業，提升數據完整性。
- **集團統一性**：除委外加工商可各自編碼外，客戶編碼與產品基本屬性需採集團統一標準。
