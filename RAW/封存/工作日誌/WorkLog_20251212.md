- **狀況概述**：
    
    - 採購部安排廠商（顧問之弟）直接參與會議。
        
    - 採購經理對於合約條款與規劃書內容理解不足，需 IT 部門逐條解釋與把關。
        
- **關鍵決議與要求**：
    
    1. **系統目標**：要求寫入合約/規劃書，系統需「全集團通用」，整合 MES、PCS、e-COA 及客戶報表，避免產線重複輸入。
        
    2. **驗收條款**：
        
        - 拒絕「10 個工作日未回覆視為驗收通過」，改為 **驗收期 1 個月** 且需 **簽名** 確認。
            
        - 拒絕「不可作為驗收作業的限制條件」條款，要求修改語意。
            
    3. **保固計算**：確定以 **驗收日** 起算（非簽約日）。
        
    4. **技術交付**：要求明列交付程式碼 (Source Code)、軟體環境建置、編譯方式；修正 Windows Server 版本描述錯誤。
        
    5. **硬體架構**：合約需明確定義 HA 架構（2 台視為 1 套）。
        
    6. **費用與訓練**：
        
        - 台灣區顧問費需事先提出；大陸出差費由採購另議。
            
        - 教育訓練日程後移，過程允許 **錄影** 供內部培訓。
            
- **顧問交辦事項**：
    
    - **廠商**：需至南廠、陸廠訪談，並產出完整規劃書供產線簽名。
        
    - **產線**：各廠需統計「在製品」整理比例（至刻號層級）。
        
    - **業務**：需將標準編碼填入在製品表格。
        
    - **IT**：要求廠商提供在製品匯入格式，並將檔案上傳網路供顧問檢視進度。
        
- **備註**：
