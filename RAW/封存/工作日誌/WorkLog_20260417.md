# 資訊部工作日誌 (2026-04-17)

## 1. 專案與開發類

* 發送「料號辨識系統」第一階段範疇確認信函（致供應商 鋒禾），要求更新 Ver.4 服務建議書。
  * 確認保固條件（一年全功能保固、12小時 SLA），並同意將「影像自動歸檔、資料庫配對與 MES 銜接」移至第二階段，第一階段專注於影像辨識本身，且暫不列入硬性辨識成功率 KPI。
  * 釐清系統資料輸出機制：要求供應商明確回覆是否提供 API 或相關介面，能即時匯出辨識成功的「料號字串」供公司現有系統拋轉使用。
  * 要求微調驗收條件與硬體規格：
    * 放寬拍照驗收限制：基於系統內建校正模組，要求取消或放寬「擺放角度端正」之限制以符合現場實況。
    * 硬體規格升級：考量多手持裝置並行傳輸與 AI 模型運算穩定度，要求將地端 AI 伺服器記憶體由 16GB 升級至 64GB。
  * **副理裁示：** 若該系統僅具備獨立輔助與手機端顯示功能，將轉由生產部吳廠長評估推動；若具備資料拋轉整合能力，資訊部才會開立請購單（PR）推進後續採購作業。
